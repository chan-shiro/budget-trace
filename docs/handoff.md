## 1. これは何か

**予算トレース** — 地方自治体の予算（歳入・歳出）から補正・執行率・支出先・事業報告までを
一次資料（エビデンス）付きでたどれる可視化サイト。
**決算は全1,741市町村**（総務省）、**当初予算（款別）は46団体**（全20政令指定都市＋山梨県内＋類似市＋
**東京特別区13区**。**新宿が H13〜R8 の26年度で最長**、次いで**文京が22年度**・中央/江東/大田/葛飾/文京が平成年度まで）、
**甲府市だけが予算→事業→成果→議会まで通っている**（他の自治体もこの深さを目指す）。

Claude Design のプロトタイプ（`project/予算トレース.dc.html`）を Next.js で本実装したもの。
`project/` は元デザインの**参照用アーカイブ**（編集・import しない）。会話ログ `chats/` は 2026-07-18 に追跡から外した（git 履歴にはある）。

**U+2015（`―`）が無く**、文京 H27・H21 の廃止款が `（特別区債）―` になっていた。

→ **ダッシュ類は1文字ずつ足さず文字クラスごと広げる**。既存217ソースを再 parse して**差分ゼロ**。

**江戸川区で H27〜R8 の12年度を収録した**（2026-07-16・§10g）。甲府（R2〜R8 の7年）を超えた

**トップに「収録の深さから選ぶ」を置いた（2026-07-16・#111）**: 事業報告まで／主な事業まで／
当初予算まで の3段に**収録済みの全団体をチップで並べ**、クリックでダッシュボードへ直行する。
**段の割り当ても件数も手書きしていない**（`MUNI_BUDGETS`・`MUNI_BUDGET_YEARS`・`REPORT_MUNIS`

ない**（横浜・川崎は主な事業一覧を持たずに事業報告を持つ）ため、階段ではなく**最も深い到達点**で
束ねている。地図・検索は従来どおり全1,741市区町村への入口として残す。

  **H30 以前は歳入が1款少なく款番号が1つずれる** → **年度をまたぐ結合は款名で行う**（款番号は不可）

  になっていた（**U+FF0D**）。**Σ は差0 で validate も素通り**＝款名の全件目視で見つけた（§2-4）。

**年号ラベルを H 対応にした（2026-07-16）**: derive・registry の `令和${fy.slice(1)}` 直書きを
`pipeline/lib/fy.ts` の `eraYear`（H/R 対応・fyRank も同居）へ集約し、クライアント側も

に集約した（**data.ts でなく別モジュールなのは data.ts → decision.ts の import があり循環するため**。
data.ts が re-export する）。**既存 gen は差分ゼロ（挙動不変のリファクタ）**。
⚠ **1回目（#110）で集約し切れておらず、5か所の直書きが残っていた** — ドリルのリクエストチップ2か所は
`budget.fy` を使うので**H 年度を入れた瞬間に「令和31年度」と出る**ところだった。**「集約した」と
書いた後こそ grep で残りを洗うこと**。あわせて validate の差引ゲートが `Number(fy.slice(1))` で
並べており **H31 が R2 より後ろに来る**（札幌の事業報告 H24〜R7 で踏む）ので fyRank に直した。

**エビデンスのリンクに例外ができた（2026-07-16）**: 「要許可」（再利用が許諾されていない）の
資料（平成年度13ソースの追加で 139資料に増えた。うち魚拓へ1＝豊島 R3）は、**写しは残した
ままリンクだけ発行元へディープリンク**する（発行元から
消えていて魚拓にしかない資料は魚拓へ）。原則（画面のリンクは常に③＝自サーバーのコピーを

### 2-2. 版面の型が出揃った（これが政令市収録の本体）

**「実測してから着手する」以外に方法が無い。**スカウトの構造報告は**毎回外れうる**
（神戸「無改修で乗る」→ 乗らない／静岡「中央寄せ3行型」→ 実際は別型／相模原「面付けで重複」→ 実態が違う）。

> **2026-07-16 追記（特別区で再発 → 手を打った）**: 千代田「見出し指定は不要」→ 実際はパーサが
> 見出しの存在を必須チェックしていて throw、など**同じ型の誤報が特別区でも3種類出た**。
> 原因は**スカウトが registry を書けずパイプラインを走らせられないため、「乗るか」を推論でしか
> 答えられなかった**こと。→ **`bun run pipeline:try-parse <parser> <file> --opts '<JSON>'`** を新設した
> （registry も data/ も触らないドライラン。抽出できた款の全件＋Σ照合を出す）。
> スカウトはこれを当てて実測で報告するようになったが、**それでも鵜呑みにしない** —
> 今回は**5体が揃って「licenseClassOf の語彙が当たらない」と誤報**し、実際に足りないのは2件だけだった
> （docs の要約を正規表現と取り違えていた）。**収録時は自分でコードとデータに当たること。**

**款別一覧の在り処（3型）**

| --- | --- | --- |

**款名の折返し（5型）**。**うち2つは幾何で見分けられないので `parserOptions` で明示する**:
1. 上段折返し（豊川・和泉）— 断片が款行の**前**
2. 中央寄せ3行（名古屋・札幌・京都款5）— 上段＋**款行(名前欄が空)**＋下段 → `awaitTail`
3. 款番号が単独行（甲府R2/R3・大阪）→ `pendNo`

5. **金額が款番号より前（上段）に来る**（静岡）→ 3と逆順。孤児を1行覚えて構造判定

### 2-3. 検証ゲートの増減（ここが一番の資産）

**足した**:
- **款名の重複 → error**。折返しの取りこぼしは金額が正しいまま款名だけ化けるので Σ を素通りする。
  失敗すると頭の断片が落ちて衝突する性質を突いた。**折返しを殺して再現確認済み**
- **廃止款を拾う**（§9c）。記号が自治体ごとに全部違う（`廃款`/`△`/`▲`/`〇`/括弧書き/**記号なし**）。
  6件の前年度Σ欠落を解消（**川崎R2 で 8.6億**）。`kanNo` は nullable（原典が振っていない番号を捏造しない）
- **Wayback の打ち切り診断**（§9b）。**MiB 境界ちょうど**で切られる。**2サンプルから「5 MiB」と定数にしたら
  1 MiB の実例が出て反証された** → 境界そのもので判定。不一致は「別版」0件・打ち切り7件
- **`/roadmap` の「進捗の数字を手書きしない」を機械化した**（2026-07-16）。3ルールのうち
  **Markdown と時期の約束しか検査していなかった**ので、**「全1,741市町村」が手書きで残っていた**
  （＝derive が `progress.muniCount` として出す値と同じ＝合併で片方だけ古くなる二重管理）。
  **数字一般は禁じない**（「札幌は R7 だけで666件」のような**外部の事実**は計画の判断材料に要る）。
  **derive が今まさに算出した値と一致する数字だけ**を落とす → 自己調整して偽陽性0・実態が動けば
  手書きが自動で検出される。**既存の違反を実際に捕まえて文言を修正した**
- **款名の破損を語彙で止めるゲート**（2026-07-16・§9m）。§2-4 の「Σ が守らない領域」を語彙で守る。
  **全224ソースを機械的に洗って本番バグを1件発見**（横浜 R3 の `一般財源国県支出金市債その他議会費`
  — gen まで到達して画面に出ていた）。挙げるのは**款名として原典にあり得ない語だけ**
  （`千円`・`一般財源`・`国県支出金`・`予算額`・`総括` 等）で、**全178ソースで偽陽性0**・
  **横浜 R3 で発火も再現確認**。⚠ **語彙の網なので万能ではない** — 未知の語は捕まらないので
  **「新資料は款名を全件目視」の規約は変わらない**
- **年度間クロスチェーンをゲート化した**（2026-07-16・§9l）。**列の取り違えを捕まえる唯一の網**を
  収録時の手作業から derive の出口チェックへ移した。**「連続年度の前年度列の合計 = 前年の当年度合計」**
  だけを見る（款ごとの差は原典の組み替えで正常に動くのでノイズになる — 実測で 125リンク中 15が
  款レベルで動き、**すべて総額は一致**した）。総額が違ってよいのは**原典が別基準と明示している
  場合だけ**（骨格予算＝`prevBasis`/`prevNote` を持つ札幌 R6・R2 と甲府 R6）。
  **足立の列順反転を実際に再現して発火を確認**（`prevColumnFirst` を落とすと **validate は4年度とも
  「ok」で通す**のに、このゲートが derive を止める）。既存125リンクではノイズ0
- **「件数だけの報告」をやめた**（2026-07-16・§9n/§9o）。**数だけの報告は「静かに壊れる」を招く** —
  同じ穴を1セッションで3つ踏んだ:
  1. archive の「新規登録 2」の裏で**台帳が丸ごと消えていた**（§9k）
  2. decision シャードの「⚠ …: 2」は**どの自治体か分からず**、しかも**どんなに大きくズレても
     警告のみで画面に出続けた**（歳出は除外するのに歳入は通す非対称）→ 名指し＋しきい値（§9n）
  3. **岡山 R2〜R4 が「魚拓はあるが sha256 を一度も照合していない」**まま「登録済み」で素通り（§9o）
  → **報告は必ず「何が」を出す。件数だけにしない。** しきい値は**分布の実測**に置く（§9b の轍）
  ⚠ **ゲートは偽陽性を出した時点で無視される** — §9o を作るとき、照合対象（pdf/xlsx/csv）と
  診断対象（全 file）を揃え忘れて **HTML 12件が毎回ノイズで出た**。**検査する対象は、
  実際に検査している対象と同じ条件にする**
- **団体コードと自治体名の同一性チェック**（§9h・下記）
- **archive の台帳を「読み直して併合」にした**（§9k・2026-07-16）。`pipeline:archive` は起動時の

  **各プロセスは「新規登録 2」と成功を報告する**ので台帳を開くまで気づけない。

### 2-4. Σ が守らない領域（繰り返し踏んでいる）

- **款名・所属名などの表示専用フィールド**（金額でないので Σ が立たない）。
  横浜「千円千円千円市税」・神戸「一般財源…議会費」・岡山「歳ゴルフ場利用税交付金」・
  仙台「交付金地方特例交付金」。**新しい資料は款名を全件目視する**（MEMORY にも記録）
- **⚠ 文字レベルで静かに壊れる型が出た**（2026-07-18・港区 §10n）: 款名が**部首の異体字**
  （Kangxi Radicals U+2F00–U+2FDF / CJK部首補助 U+2E80–U+2EFF）で組まれる。`⺠⽣費` は `民生費` と
  **見た目がほぼ同じ**で、**Σ も語彙ゲートも通る**。目視でも見落としうる。→ **validate に部首レンジの
  error ゲートを足し**（`KANNAME_RADICAL_RE`）、**パーサに `fixRadicals`**（NFKC＋U+2E80台の明示対応）を
  入れた。**款名の目視スクリプトには部首レンジのチェックを必ず含める**（`0x2E80<=ord(c)<=0x2FDF`）。
- **主な事業の Σ ≤ 款予算**は**抜粋なので通ってしまう**。相模原で**新規の親事業を36件まるごと
  落としていた**のに素通りした → **「マーカーの数 = 拾えた事業の数」の assert が唯一の網**

### 2-5. 積み残し・注意

- **`data/raw` が 829MB**（政令市14市＋特別区13区で急増。文京の22年度で +9MB）。
  **2026-07-17 に測って「git からは出さない」と決めた**（下記）。**再検討させないために経緯を残す**:
  - **git 履歴は 1.21GB で、うち 0.96GB（79%）が今も HEAD で生きている**。PDF は一度置いたら
    書き換わらないので、**blob を git から追い出す通常の動機（履歴の churn 膨張）が当たらない**。
    GitHub の上限 5GB に対して24%。最大ブロブは 26.8MB（100MB 制限にも余裕）

    **raw はビルドの入力**なので、どこに置いてもビルド時にバイトが要る。「clone 1.2GB」が
    「R2 から 829MB DL」に変わるだけで総量は減らない。**LFS は毎ビルドで帯域を食う**
    （無料枠 1GB/月）ので**むしろ悪化**する。加えて raw を git に置くのは①（原本アーカイブ）
    そのもので、外に出すと3層が脚を1本失う
  - **効いたのは③の配信を絞ること**（下記）。**git は触っていない**
- **③の配信を要許可の資料に限らなくした（2026-07-17）**: リンクが発行元へ振り替わる資料は
  コピーを置いても**誰も開けない**。**829MB 中 667MB（80%）が到達不能なまま配られていた**ので
  `sync-public-sources` が `RESTRICTED_EVIDENCE` を見て外す（**829MB → 162MB**）。

## 4. 横断的な暗黙知（ハマりどころ）

- **依存は Next 16 + React 19（2026-07-14 更新）／`bun audit` 0 件を維持する**。更新前は Next 14 系で
  **17件（high 7）** の既知脆弱性があった（Server Components の DoS・WebSocket SSRF 等。next ≥15.5.16 で修正）。
  Next はおおむね直近2メジャーしか修正が出ないので、**メジャー遅れを放置しない**。ハマりどころ:
  - **xlsx は `pipeline/lib/xlsx.ts` 経由でのみ使う**（`"xlsx"` を直接 import しない）。SheetJS は 0.18.5 で
    npm 公開を停止 → 公式 CDN 版を devDependency で導入（`https://cdn.sheetjs.com/xlsx-0.20.3/...tgz`）。
    ESM ビルドは `readFile` が fs を掴まず "Cannot access file …" で落ちるため、ラッパで `set_fs` 済み。
    **アプリ本体は xlsx を使わない（パイプライン専用）**ので devDependencies が正しい
  - `postcss` は Next が 8.4.31 固定 → `overrides` で 8.x 最新へ（GHSA-qx2v-qp2m-jg93）
  - React 19: グローバル `JSX` 名前空間が廃止 → `React.JSX`。`Record<string, unknown>` と交差した props は
    既知プロパティが `unknown`/`{}` に落ちるので使う分だけ絞り込む（`ui.tsx` の HoverBox）
  - Next 15+: `params`/`searchParams` は Promise → catch-all ルートで `await`
  - TypeScript は 5.5 のまま（7 系は Go 実装で飛躍が大きく、脆弱性指摘も無いため別途）
- **ブラウザ検証ツールの限界**: プレビュー用ヘッドレスブラウザは**コンポジットしない**ため
  (a) スクリーンショットが body 背景だけの空画像になる (b) viewport が 0×0 になることがある
  (c) **IntersectionObserver のコールバックが一切発火しない**（自明に可視な要素でも）。
  → **PdfViewer の遅延描画（IO でページを描く）はこのツールでは検証できない**（PDF.js の読込・
  ページ数・ビューポート計算・初期スクロールまでは確認可能）。DOM 検証（`get_page_text` /
  `getBoundingClientRect` / computed style）は信頼できるので、そちらで確認する
  - **(d) `requestAnimationFrame` も発火しない**（2026-07-16 に実測。rAF を待つ Promise が
    タイムアウトする）。→ **`CountUpNum` の数値はこのツールでは常に `0万円` に見える**
    （`useState(0)` から rAF でカウントアップする実装のため）。**これはバグではない** —
    ダッシュボードの「歳入合計/歳出合計」と総額だけが CountUpNum で、**款別の内訳は静的なので
    正しい値が読める**。総額を検証したいときは内訳か gen 側の値で確認すること。
    なお `prefers-reduced-motion: reduce` なら即座に確定値を出すので、実ユーザーには影響しない

- **GitHub 運用**: **`main` へ直接 push しない**（main push = 本番デプロイ。必ず作業ブランチ → PR →
  squash マージ。「マージしよう」は PR を作れの意）。このリポジトリは **squash マージが使われる**。マージ後は必ず
  `git fetch origin main && git checkout -B implement-budget-trace origin/main` でブランチを作り直す
  （古いコミットを積み直すと重複する）
- **データ来歴**: 画面の「ダミー」「推計」注記は仕様。データを触るとき整合を崩さない。
  フィクスチャ出力（`data/normalized/_fixtures/`）をアプリから import しない
- **PALETTE はモジュール変数**: `src/client/lib/data.ts` の `setPalette()` で描画前に切替。呼び出し順に注意
- **スタイルは `S("...")` の CSS 文字列**（プロトタイプと diff 可能な形を維持）。
  ホバーは `HoverBox`、モバイルは `data-mq` 属性 + `src/app/globals.css` の media query
- **JapanMap は React 管理外の直接 DOM 操作を含む**（`boxRef` 配下の SVG）。境界を崩さない
- **地図データは `public/mapdata/` に同梱**。CDN 実行時フェッチに戻さない
- **政府系サイト（総務省・e-Stat）はクラウド環境から 403**（環境の許可リスト + 先方 WAF の2層）。
  ローカルなら制限なし（fetch.ts はブラウザ相当 UA を名乗る。R6 は自動取得できた）。
  クラウドで通す場合は環境設定で `*.soumu.go.jp` 等を Custom 許可
  （「Also include default list …」に必ずチェック。外すと git/bun が壊れる）
- **決算状況調の実ファイルの構造**（R6 で確認。パーサ 0.2.0 が対応済み）:
  1年度 = 4ファイル（都市別/町村別 × 概況/目的別歳出内訳）で、団体コードでマージする。
  ヘッダは複数行（目的別は 款番号/科目名・連番/内訳名 の3段＋結合セル）。
  労働費・諸支出金は総額列がなく内訳列の合算。「（参考）一般行政経費」ブロックは
  款名を再掲するので列解決から除外。都道府県名の列はなく「団体コード空 + 団体名だけ」の
  区切り行（字間スペース入り）で県が変わる。特別区は都市別ファイルに含まれる
- **描画用値オブジェクト `v` は any のまま**（プロトタイプ移植由来・eslint-disable 付き）。
  新規コードでは倣わない。サーバー層導入時に型付けして解消する予定
- IME 変換確定 Enter のガード（CLAUDE.md 参照）— Enter ハンドラ追加時は必須
- **`pdftotext` の使い分け**（2026-07-16 に全部踏んだ）:
  - `-layout` が既定。ただし**縦書き・面付け・左右2側**があると文字が交互に混ざる（相模原 `歳歳入出`）
  - **`-layout` の表示列は当てにならない**場面がある — 款項目の文字が前に詰まると**同じ階層の行が
    別の列に見える**（相模原の主な事業は親の `○` が列27・内訳の `○` が列22 で**逆転**）。
    **階層を x で判定するなら `-tsv`（座標）を使う**。`pdfPageWords()` が既にある
  - `-x/-W` で**ページを横に切れる**（静岡）。座標を自前で組み直すより先にこれを試す
  - **`-bbox-layout` は poppler が `std::out_of_range` で落ちるファイルがある**（静岡 R5・新潟）
- **zsh は変数を単語分割しない**（このセッションで3回踏んだ）。`for t in $TARGETS` は**1回しか回らない**。
  `printf '%s\n' a b c | while read -r x` を使う
- **同じ字に見えて別の文字**が繰り返し出る: `○`(U+25CB) と **`〇`(U+3007)**（北九州の廃止款・
  相模原の事業マーカーに混在）、`−`(U+2212) と `-`。**片方だけ数えると件数が合わない**
- **`data/raw` は 475MB**。政令市を1市足すと数MB〜25MB 増える。**Wayback は MiB 境界で打ち切る**ので
  大きいファイルは②層が張れない（神戸R8 24MB→5MiB・浜松R5 3.9MB→1MiB）
